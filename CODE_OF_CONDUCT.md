# KBeacon Code of Conduct

KBeacon is an open source community project. We are committed to a welcoming, respectful, inclusive, and harassment-free collaboration space for everyone.

## Expected behavior

Participants are expected to:

- be respectful and professional;
- assume good intent while still asking clarifying questions;
- give and receive constructive feedback;
- focus disagreement on ideas, code, design, documentation, and user impact;
- respect different experience levels, backgrounds, time zones, and communication styles;
- help keep issues, pull requests, discussions, and community spaces productive.

## Unacceptable behavior

Unacceptable behavior includes:

- harassment, intimidation, threats, or sustained disruption;
- discriminatory language or behavior;
- personal attacks, insults, or trolling;
- publishing private information without consent;
- sexualized language or imagery in project spaces;
- repeated off-topic or bad-faith participation;
- any behavior that would make the project unsafe for contributors or users.

## Scope

This Code of Conduct applies in all KBeacon project spaces, including GitHub issues, pull requests, discussions, documentation, examples, release workflows, community chats, and events where someone represents the project.

## Reporting

Report Code of Conduct concerns privately to the maintainers listed in `MAINTAINERS.md`.

Do not use public GitHub issues for sensitive reports.

## Enforcement

Maintainers may take action to protect the community, including:

- asking for clarification or edits;
- removing comments;
- closing issues or pull requests;
- limiting participation;
- blocking or reporting abusive accounts.

Enforcement decisions should be made with care, proportionality, and a focus on community safety.
